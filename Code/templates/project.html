IQR CAPPING

Interquartile Range (IQR) capping is a technique used in statistics to handle outliers. Outliers are data points that are significantly different from other observations. They can affect the results of the data analysis and statistical modeling in a negative way by skewing the results and decreasing the accuracy of the predictions. The Interquartile Range (IQR) is a measure of statistical dispersion, or in simpler terms, it's a measure of how spread out the values in a data set are. It's calculated as the difference between the 75th percentile (Q3) and the 25th percentile (Q1) of the data. IQR capping, also known as "IQR outlier removal" or "outlier capping", is a technique used to remove outliers from the data. Here's how it works: 1. Calculate the IQR of the data. 2. Define the lower bound as Q1 - 1.5 * IQR and the upper bound as Q3 + 1.5 * IQR. 3. Any data point that falls below the lower bound or above the upper bound is considered an outlier. 4. These outliers are then capped to the lower and upper bounds, meaning that any value below the lower bound is set to the lower bound, and any value above the upper bound is set to the upper bound. This method is effective because it only removes extreme values and it uses the IQR, which is robust against outliers. However, like any other method, it should be used with caution because if not used properly, it can lead to loss of information or biased results.

Clustering Model Selection Using 3 Different Metrics

The Silhouette Score is a measure of how similar an object is to its own cluster compared to other clusters. The best value is 1 and the worst value is -1. The Calinski-Harabasz Index is a ratio of the sum of between-clusters dispersion and of inter-cluster dispersion for all clusters (where dispersion is defined as the sum of distances squared). The higher the value, the better. The Davies-Bouldin Index signifies the average ‘similarity’ between clusters, where the similarity is a measure that compares the distance between clusters with the size of the clusters themselves. Zero is the lowest possible score.

Tuning The Selected Model

Improve Model Performance: The primary reason for hyperparameter tuning is to find the optimal combination of hyperparameters that minimizes a predefined loss function to give the best possible performance of the model. Prevent Overfitting and Underfitting: Hyperparameter tuning helps in finding a balance between bias and variance. A model with high bias (underfitting) oversimplifies the model and performs poorly on both training and test data. A model with high variance (overfitting) overcomplicates the model and performs well on training data but poorly on unseen data. Tuning hyperparameters like regularization parameters can help prevent both overfitting and underfitting. Optimize Resources: Some hyperparameters control how much computational resources the model will use, like the number of hidden layers or units in a neural network, or the depth of a decision tree. By tuning these, we can find a good trade-off between computational cost and model performance. Better Understanding of Model and Problem: The process of hyperparameter tuning can provide valuable insights into the dynamics of the model and its relationship with the underlying problem. It can help us understand which hyperparameters are most important, how they interact, and what values make sense.
